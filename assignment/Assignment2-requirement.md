# Assignment 2 - Advanced Website based on Modern Front-end Libraries

This is a group assignment in which students demonstrate the use of modern frontend libraries (e.g., React, Vue.js, Angular, Svelte) to design and implement a relatively complex dynamic website for a real-world application. You need to incorporate an appropriate backend (e.g., FastAPI, Node.js, Flask) to work with a database (e.g., MySQL, MongoDB). Alternatively, you may use full-stack frameworks (e.g., Django, Laravel, Spring Boot) for this assignment without being limited to the FARM stack (FastAPI, React, MongoDB).

The group component and the individual component account for 15% and 25% of the subject's total mark, respectively. The group size is ideally 2 or 3, and should be no larger than 5 (exceptions apply). For those who choose to complete this assignment individually, the group component is treated as an individual component.

## Project Topic

You may propose/choose a new project, or extend your work from Assignment 1 by involving at least three types of entities to which CRUD operations could be applied:

## E-commerce Shopping Cart 
Registration/login: User authentication using password hashing and JWT (user).
Live search: A search bar that filters the product list in real-time as the user types (product).
User profile: An admin can view all users' shopping carts (shopping_cart).

## Requirements

Your app should be (or at least behave like) a single-page application. Ideally, there should be only one .html file used in the project, and your app should dynamically rewrite the current page with new data rather than reloading a new page from the server.
Your app should involve all CRUD (Create, Read, Update, Delete) operations on a database.
 Your app should have a seamless and streamlined interface. The user shouldn't experience interrupted interactions or unnecessary steps when completing a task.
Your proposed project should be at least comparable in complexity to the three choices above.
